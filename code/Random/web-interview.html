DOM Manipulation

  • The DOM (Document Object Model) is one of the most fundamental components of any web application.
      • Important concepts to know about the DOM include: reading, writing, and modifying the DOM. As well as accessing and removing elements within the DOM.

  • Understanding the DOM

    DOM structure
      • The DOM is the structure that gives you access to all the content within a website. These can be targeted through JavaScript, CSS rules, and more.

    DOM nodes
      • The key to accessing the elements within the DOM are nodes. Think of nodes as a branch of a tree that has different sub-branches. You can access the content of your page by targeting the proper node.

    Targeting nodes
      • To start targeting nodes, you need to start by targeting the window element in the current document, which is the root of the entire DOM, and then asking for a node that matches your criteria.
      • JavaScript will return a specific node, or an array-like object with the nodes that you need.
      • The most popular way to access DOM nodes is getElementById(). This returns an element if that element has an ID attribute that you specify, but only if you specify an ID for that element. This is the oldest and most compatible of old DOM selection methods.
      • There are now methods that allow you to grab elements by either class name or tag name. These are getElementsByClassName() and getElementsByTagName(). The difference is that these will return a list-like array of nodes, and you'll need to loop through them to get individual elements.

    Modern Targeting Methods
      • All developers should be aware of the querySelector and querySelectorAllmethods.
      • querySelector returns the first matching element
      • querySelectorAll returns a list-like object of all the elements that match the criteria.
      • Remember that these are accessed natively in JS through the DOM document object (document.querySelector()).

    Manage nodes
      • Now that you can select nodes, it's up to you on how you want to add, modify, or delete content within the nodes.
      • You can append regular HTML text to a node using the innerHTML property.
      • You can create nodes manually using createElement().
      • You can insert a node inside another node using appendChild().
      • You can add events to nodes using the addEventListener() method.

                '#6D73C2', //purple

            function makeBoxes(howMany) {
                var colorAmt = colors.length;
                var currColor = 0;

                var myElement;
                var myNode = document.querySelector('.boxes');

                for(var i = 0; i < howMany; i++) {

                    myElement = document.createElement('div');
                    myElement.className = 'box';
                    myElement.style = 'background-color:' + colors[currColor];

                    myNode.appendChild(myElement);

                    if(currColor == colorAmt-1) {
                        currColor = 0;
                    } else {
                        currColor++;
                    }
                }

                myNode.addEventListener('click', function(e) {
                    e.target.parentNode.removeChild(e.target);
                }, false);
            }

Bringing Closures

  • What to know before: Scope

    Scope refers to the area within a computer program where a variable is available.
      • Variables are created and exist within different regions.
      • Variables are bound to certain regions within your application.

    Javascript uses a concept called lexical scoping.
      • Lexical scope describes how variable scopes are determined when there is a function inside another function.
      • Using lexical scope, the inner function will have access to variables in the parent function. (This is essentially what a closure is.)

    A closure is created when you use the function keyword inside another function.
      • This means that variables can remember the environment that they were created in.

    Scopes & Closures -
      • Scope region binding
      • Lexical scoping
      • Creating a closure
      • Remembers environment

  • Simple example

                function setupItem() {

                var myNumber = 1;

                function increment() {
                    return console.log(myNumber++);
                }

                increment(); // 1
                increment(); // 2
            }

            setupItem();

    The interesting thing happening here is that the inner function is going to keep remembering the value of the number, even though it's changing as the function runs different copies.

  • Closures in Objects

    Closures are important for objects because they can remember the state.

                function myObject() {
                var myValue = 1;
                return  {
                    display: function() {
                        return console.log(myValue);
                    },
                    increment: function() {
                        return myValue++;
                    }
                };
            }

            var mything = myObject();
                mything.display();  // 1
                mything.increment();
                mything.display(); // 2

            var otherthing = myObject();
                otherthing.display(); // 1
                otherthing.increment();
                otherthing.display(); // 2

  • Closures and Loops

    It's important to be careful when using closures inside loops, because closures remember the value of the environment and those variables are constantly changing.
      • A technique for getting around loops in closures is called a function factory.
      • A function factory is a technique where you create a loop to create a series of functions.
      • This works very well for creating a series of events.

IIFE

  • IIFE - Immediately-invoked Function Expression

                (function() {
                //Isolate Code Here
            })();

    An IIFE is what's known as an anonymous function, which is simply a function with no name. Sometimes IIFEs are called anonymous closures.
      • This is a function that we want to invoke immediately.
      • By making this function an expression, it can then be executed immediately. Which is done with the extra set of parentheses at the end to execute it.

  • Protection

    Typically you'll want to wrap an entire script in an IIFE which will provide some special features.
      • This will protect any variables or functions inside this master function from being accessed.
      • Functions inside this IFFE are going to create closures.
      • Therefore, any variables declared in the IFFE will be accessible to any function that is also written in the IFFE, and protected from anything outside the IFFE.

  • Example

                //IIFE
            (function() {
                var one = 1;
                console.log(one); // 1
            })();

            console.log(one); // ReferenceError: one is not defined

Absolute vs. Relative Positioning

  • One of the most important properties a page element can have is the position property.
      • It's used controlling how elements appear in relation to other elements.
      • There are 4 main position properties available; static (default), relative, absolute and fixed (sticky-experimental).
      • Absolute and Fixed positions are tricky.
      • Main thing to remember here is that position property control how elements affect the flow of content.

    A key difference is whether the position is adjustable.
      • This is done using additional properties: top, right, bottom, or left (TRDL)

    Another key difference is whether the property removes the current element from the flow of the document.
      • If removed, then the block containing the element will behave as if it's not there.

    A final key difference is how the change in position appears in relation to different elements on the page.
      • It can either be aligned to another element, or be relative to the entire document.

  • Options

    static relative absolute fixed
    Adjustable no yes yes yes
    Flow normal normal removed removed
    Relative to n/a current parent/document document

  • A good example of using this property is implementing a tool tip. Say we have a block, and a tooltip.
      • The goal is to have the tooltip show up relative to the block.
      • The block would need to have a position of relative.
      • Then if the tooltip is set to absolute, it wil be displayed where it's parent (block) is.
      • You can then use to TLDR to adjust it's position relative to it's parent element.
      • Then you can use tricks like hover and display none to only get the tooltip to display when the button is hovered.

Callbacks

  • A callback is a convention in JavaScript that is often used with asynchronous programming.
      • A callback is typically used to do things asynchronously, or whenever you want to execut code at a certain point in time or when an event takes place.
      • They are often tied to events to take care of tasks whenever the events are triggered.
      • Callbacks use the anonymous function (function with no name), because they are invoked immediately once the event takes place.

  • Example

                var tips = document.querySelectorAll('.hastip');

            for(var i = 0; i < tips.length; i++) {

                //Toggles the active class on a clicked tooltip. 
                tips[i].addEventListener('click', function(e) {
                    e.target.querySelector('.tooltip').classList.toggle('active');
                }, false)

            }

Flexbox - Control Horizontal Alignment

  • Flexbox is one of the available CSS display properties and it makes single-dimension layouts super easy.
      • It's an important tool in a front-end developers toolkit.
      • CSS Grid is a good alternative for complex layouts, but does not have great browser support.
      • It's recommended to use some post-processing technology to manage conversion to different versions of syntax for different browsers.

  • How To

    You specify the display property on a parent element, and then control how a child element will display within that parent.
      • Generally, this means you'll be adding additional properties in the parent to control how the children are displayed.
      • There are also properties that you can implement on individual children within the parent, but most of the time, the changes are made on the whole parent.

    The main task you need to do is set the parent to flex.
      • This force the children to start behaving as a flex component.

                .parent {
                display: flex;
            }

    Once parents are acting as Flexbox components, you can specify how elements inside will flow.
      • The default is row: elements will display side-by-side.
      • Another option is column: elements will stack on top of one another.
      • The other two options are row-reverse and column-revers: elements will be in the reverse order of their rows or columns.

                .parent {
                display: flex;
                flex-direction: DIR; //row column row-reverse column-reverse
            }

    Flex children will try to stay in a single line, but you can change that.
      • Items can be set to wrap or not wrap.
      • Items can be reversed with wrap-reverse.

                .parent {
                display: flex;
                flex-direction: row;
                flex-wrap: WRP; //nowrap wrap wrap-reverse
            }

    Finally, you can justify content to control how elements are aligned within the main axis.
      • The default is for them to be left aligned which is called flex start.
      • Elements can be easily right aligned with flex end.
      • A self-explanatory way of centering elements is using center.
      • In order to place the content that's at the beginning or the end of the parent at the edges of the container, use space-between.
      • In order to place equal space between all the elements as well as in the edges, use space-around.

                .parent {
                display: flex;
                flex-direction: row;
                flex-wrap: nowrap;
                justify-content: JUS; //flex-start flex-end center space-between space-around
            }
