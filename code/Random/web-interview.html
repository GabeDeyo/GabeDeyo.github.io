DOM Manipulation

  • The DOM (Document Object Model) is one of the most fundamental components of any web application.
      • Important concepts to know about the DOM include: reading, writing, and modifying the DOM. As well as accessing and removing elements within the DOM.

  • Understanding the DOM

    DOM structure
      • The DOM is the structure that gives you access to all the content within a website. These can be targeted through JavaScript, CSS rules, and more.

    DOM nodes
      • The key to accessing the elements within the DOM are nodes. Think of nodes as a branch of a tree that has different sub-branches. You can access the content of your page by targeting the proper node.

    Targeting nodes
      • To start targeting nodes, you need to start by targeting the window element in the current document, which is the root of the entire DOM, and then asking for a node that matches your criteria.
      • JavaScript will return a specific node, or an array-like object with the nodes that you need.
      • The most popular way to access DOM nodes is getElementById(). This returns an element if that element has an ID attribute that you specify, but only if you specify an ID for that element. This is the oldest and most compatible of old DOM selection methods.
      • There are now methods that allow you to grab elements by either class name or tag name. These are getElementsByClassName() and getElementsByTagName(). The difference is that these will return a list-like array of nodes, and you'll need to loop through them to get individual elements.

    Modern Targeting Methods
      • All developers should be aware of the querySelector and querySelectorAllmethods.
      • querySelector returns the first matching element
      • querySelectorAll returns a list-like object of all the elements that match the criteria.
      • Remember that these are accessed natively in JS through the DOM document object (document.querySelector()).

    Manage nodes
      • Now that you can select nodes, it's up to you on how you want to add, modify, or delete content within the nodes.
      • You can append regular HTML text to a node using the innerHTML property.
      • You can create nodes manually using createElement().
      • You can insert a node inside another node using appendChild().
      • You can add events to nodes using the addEventListener() method.

                var colorAmt = colors.length;
                var currColor = 0;

                var myElement;

                    myElement = document.createElement('div');

                        currColor = 0;
                    } else {
                        currColor++;
                    }
                }

                    e.target.parentNode.removeChild(e.target);
                }, false);
            }

Bringing Closures

  • What to know before: Scope

    Scope refers to the area within a computer program where a variable is available.
      • Variables are created and exist within different regions.
      • Variables are bound to certain regions within your application.

    Javascript uses a concept called lexical scoping.
      • Lexical scope describes how variable scopes are determined when there is a function inside another function.
      • Using lexical scope, the inner function will have access to variables in the parent function. (This is essentially what a closure is.)

    A closure is created when you use the function keyword inside another function.
      • This means that variables can remember the environment that they were created in.

    Scopes & Closures -
      • Scope region binding
      • Lexical scoping
      • Creating a closure
      • Remembers environment

  • Simple example

                function setupItem() {

                var myNumber = 1;

                function increment() {
                    return console.log(myNumber++);
                }

                increment(); // 1
                increment(); // 2
            }

            setupItem();

    The interesting thing happening here is that the inner function is going to keep remembering the value of the number, even though it's changing as the function runs different copies.

  • Closures in Objects

    Closures are important for objects because they can remember the state.

                function myObject() {
                var myValue = 1;
                return  {
                    display: function() {
                        return console.log(myValue);
                    },
                    increment: function() {
                        return myValue++;
                    }
                };
            }

            var mything = myObject();
                mything.display();  // 1
                mything.increment();
                mything.display(); // 2

            var otherthing = myObject();
                otherthing.display(); // 1
                otherthing.increment();
                otherthing.display(); // 2

  • Closures and Loops

    It's important to be careful when using closures inside loops, because closures remember the value of the environment and those variables are constantly changing.
      • A technique for getting around loops in closures is called a function factory.
      • A function factory is a technique where you create a loop to create a series of functions.
      • This works very well for creating a series of events.

                (function() {

    An IIFE is what's known as an anonymous function.
      • This is a function that we want to invoke immediately.

  • Protection

      • Functions inside this IFFE are going to create closures.
