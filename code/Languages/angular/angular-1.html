Introduction

  • What is AngularJS?

    JavaScript Framework - takes advantage of built-in features, in this case; templating, filters, two-way data-binding and more.

    Designed to allow you to create single page applications - so regardless of the size of your application, your browser is going to load a single page. Additional content will be loaded into the application as needed. The site only loads content it needs, instead of an entire page.

  • Why use AngularJS?

    AngularJS features an excellent templating engine. Templating let's you write HTML templates with special tags, and Angular will replace those tags with data.

    AngularJS handles DOM masterfully - data is directly bound to your template. Therefore, pulling in data or interacting with your elements lets you populate your html.

    AngularJS includes commands for loading external data by including AJAX functionality. Therefore, we can use a Model-View-Controller architecture to focus more on an object-oriented approach.

  • MVC Architecture

    Model - Your data is known as a model. This can come from an internal data structure, an external JSON document or even a database.

    View - The way your models/data are displayed are your view. Typically done through a template.

    Controller - The glue that ties your data (models) to your templates (views).

  • Directives

    Another core feature behind AngularJS is something sprinkled in your HTML called Directives.

    Directives are commands that tell Angular to do its job.

    They all will start with "ng" followed by their specific command. (i.e. ng-app, ng-view, ng-model)

  • Data-Binding

    Templating is possible because of a feature known as data-binding.

    The goal is to create placeholders for content that get linked to data information.

    This also enables us to use form elements that can be tied to the data.

  • Filtering

    AngularJS has a filtering language so that data can be organized in a specific way automatically or easily change by form elements.

  • Modules

    Instead of creating a single, large JS doc, you can break up functionality into modules.

    Modules make code easier to maintain and distribute.

  • Routes

    Routes lie at the heart of single page applications.

    Routes divide your templates so that functionality behind your app can exist in different places.

  • Controllers

    All of these details are tied together through controllers.

    Controllers will help us maintain the functionality of our views and models.

Modules & Scope

  • One key concept of the MVC Architecture is that you can organize large applications into modules, separating functionality into smaller pieces of JavaScript.

    We do this in Angular using different controllers for our app.

  • Let's create a Module in a new js file called js/controller.js. Make sure to include this script.

    The variable myApp is going to have all the code for our application. This is called name-spacing. It's away to protect our application code and make sure that no other script on your website is going to interfere with our application.

                var myApp = angular.module('myApp', []);

    Then in the initial html file, we're going to add this to the ng-app directive:

                < lang="en" ng-app="myApp">

    That directive tells Angular the code for whatever is inside the HTML tag, which in this case is the entire page, is going to be controlled by this new myApp module.

  • Now let's get started with a Controller. 

                var myApp = angular.module('myApp', []);

            myApp.controller('MyController', function MyController($scope) {
                $scope.author = {
                    'name'    : 'Gabe Deyo',
                    'title'   : 'Web Developer',
                    'company' : 'IBI Data'
                }
            });

    $scope will be covered more later, but know now, it's like a super variable that you can use to pass things from your JS to your application and template, and vice-versa.

    Now we need to go back and add this controller to our template.

                <body>
                <div ng-controller="MyController">
                    
    {{ author.name }}
    
                    
    {{ author.title + ', ' + author.company }}
    
                </div>
            </body>
