Introduction

  • The ability to read data and communicate information asynchronously is fundamental to modern web applications. In this section, we'll tak a look at integrating AJAX (Asynchronous JavaScript and XML) with JavaScript and jQuery - a combination that makes it easy to work with dynamic data and reduces the number of requests to the server. The section first explains what AJAX is, and shows how to load and reload pages with pure JavaScript, update the DOM, and read and parse data in the XML and JSON formats. Next, learn how to read and write information between the server and the client with AJAX methods in jQuery, and build a dynamic search application that updates search results as you type.

AJAX

  • How do browsers work?

    • A browser requests a page
    • Client requests info from the server
    • Users sometimes request additional info
      • Without AJAX: Receive entire page again, with additional info
      • With AJAX: Server only sends additional info, only needs to update new info

  • How AJAX works

    AJAX is based on several technologies working together: Asynchronous JavaScript and XML
      • Requests without reload
      • JavaScript for heavy lifting
      • Talks to server through XHR

    What XHR isn't
      • Not XML
      • AJAX requests can be in any format (Text file, HTML, JSON object, etc.)

Using a Synchronous XHR request

  • The first step to working with AJAX is learning the API browsers provide for sending/receiving information.
      • This is done using an XML HTTP request or XHR object.

  • Example

    Lets say we have and index.html, a data.txt, and a script.js all on the same directory.
      • We want to create the simplest XHR request.

                //script.js

            var request = new XMLHttpRequest();
            request.open('GET', 'data.txt', false);
            request.send();
            if(request.status === 200) {
                console.log('Request OK');
                document.writeln(request.dataInFile); // Writes data from file in document
            }

    Note that this XHR request is synchronous, therefore, the document can't update until all the XHR requests have been made.
    • Our code will work fine in this case, but only because we're only making one request. Let's try something ugly.

                //script.js
            for(var i = 0; i < 100; i++) {
                var request = new XMLHttpRequest();
                request.open('GET', 'data.txt', false);
                request.send();
                if(request.status === 200) {
                    console.log('Request OK');
                    document.writeln(request.dataInFile); // Writes data from file in document
                }
            }

    This call is now getting made 100 times before allowing the document to update. This results in serious wait-time.

  • Since AJAX requests are supposed to be asynchronous, they should be running in the background independently of other events.
