Preprocessor

  • • The C processor provides a number of essential capabilities for the C++ language.
    • The preprocessor is an essential part of the compiler tool chain, and understanding how it works and what can be used for is an essential part of understanding C++.
    • In order for you source code to become an executable program, it needs to be compiled first.
    • This process is really a combination of several processes. Preprocessor, Compiler, Optimizer, and Linker.
    • Each of these steps is separate and distinct, even though they're often invoked with one command.

  • • The C++ preprocessor is borrowed from C. In fact, most compilers use the same preprocessor for both languages.
    • The preprocessor's job is to process the source code before, and to prepare the source code for the compiler.
    • On of the more common uses for the processor is file inclusion.
    • When you use the include directive, your source will be treated as if the entire contents of the file you name here, were included in place of the include directive.

  • • Once all of the include directives have been processed, the combined result is treated as one file called a translation unit.
    • A preprocessor macro works like a really smart alias. Ex, after this declaration: 

    #define ONE 1
    any use of the word "ONE" in all caps, will be replaced by the digit "1". No math is performed, this is really just a string substitution in the source file.

  • • Let's see another example: 

    #define MAX(a, b) (a > b ? a : b)

    • In this second example, we have a macro with parameters. Those parameters may be used in the replacement.
    • This usage is common in C, but less so in C++, where templates are often used instead.

  • • Lets see a popular example: 

                #ifndef CONDITIONAL_H_H
            #define CONDITIONAL_H_

            #define NUMBER 2
            #define CONSTANT 73

            #endif /* CONDITIONAL_H_ */

    • Conditional Processing allows you to compile parts of your code only if certain conditionas are true.
    • In the conditional example, the contents of a header file are prevented from processing more than once. (From #ifndef to #endif)
    • This technique is often called an include guard.

  • • Lastly we'll see a couple more lines of code: 

                #pragma long_calls
            #pragma mark Label
            #pragma align 32

    • Pragmas may be used to define implementation-specific behaviors for the compiler.
    • Most common uses of pragmas relate to supporting specific compilers and computing architectures.

Macros as Constants

  • • One common use for the preprocessor is to define constants. Strictly speaking, they're macros.
    • Preprocessor constants are not actual constants, although they're commonly called constants, and are used for that purpose.
    • They're commonly used as constants, but there are some imporant distinctions.

  • • Preprocessor constants are typically defined with the pound sign at the very first position in the line.
    • Then followed by the word define, and the symbol and what it exapands to.
    • So if we create a preprocessor constant: 

    #define ONE 1

    • When the preprocessor encounters this symbol (ONE), it actually replaces it with the literal value before the compiler ever gets to it.
    • Notice: This is not a variable; we can't get it's address, it won't show in the debugger, you can't use a pointer with it. It gets replaced with a literal number one before the compiler ever gets to it.

  • • Keep with tradition by defining preprocessor constants using all capital letters.
    • It's important to realize that these are not constant variables, rather they're simple text replacements.
    • Its' often better to use variables declared with the constant keyword.

Including Files

  • • The include directive is used to include a header file on your source code.
    • Essentially, the preprocessor replaces the include directive with the entire contents of the included header file then passes that combined file to the compiler.
    • Multiple include directives are allowed and in fact common. They can usually be found nested within include files.

  • Angled Includes

    #include <cstio>

    This form is used for system level headers that are common to different projects and source files.

  • Quoted Includes

    #include "stdafx.h"

    This form is commonly used for header files specific to a given project or source file.

Conditional Compilation

  • • The preprocessor provides a number of directives for conditional compilation.
    • Typically used for separate versions of code, for debugging, or for supporting different target environments.
    • #if, #ifdef -> if defined
    • #ifndef -> if not defined
    • #else, #elif -> else if and end if
    • #if defined(MACRO), #if !defined(MACRO) == #ifdef and #ifndef

  • Example 

                #ifndef CONDITIONAL_H_
            #define CONDITIONAL_H_

            #ifdef FOO
            #define NUMBER 47
            #else
            #define NUMBER 2
            #endif
            #endif //CONDITIONAL_H_

Defining Macros

  • • The preprocessor provides a powerful and flexible macro system.
    • We've already covered constant macros in previous sections.
    • You can give macros parameters, and preform operations with the inputs.

  • Example 

                //No space between MACRO and it's parameters!
            #define MACRO(a, b) ( a * b )

            int main() {
                int ia = 5;
                int ib = 7;
                printf("value is %d\n", MACRO(ia, ib)); // 35
            }

  • Max Example 

                #define MAX(a, b) (a > b ? a : b)

            int main() {
                int ia = 5;
                int ib = 7;
                printf("value is %d\n", MAX(ia, ib)); // 7
            }

  • • Because Macros are expanded to text before the compiler ever sees it, there's a potential for side effects.
    • For many uses, C++ templates are a superior solution.
    • Parameterized macros can work fine, but generally speaking, templates are recommended over parameterized macros.
    • Preprocessor Macros can be very useful, but there's also a real possibility of unintended results.
    • One common cause of problems is parameters being evaluated more than once.

Line Continuation

  • • Line Continuation in C++ allows you to use backslash as an editor escape character.
    • Perfect for doing multi-line statements with macros.

  • Non-Working Example 

                //Errors
            #define SWAP(a, b) do {
                    a ^= b;
                    b ^= a;
                    a ^= b;
                } while(0)

  • Fixed Example 

                //No Errors
            #define SWAP(a, b) do {  /
                            a ^= b;  /
                            b ^= a;  /
                            a ^= b;  /
                        } while(0)

Include Files Only Once

  • • When a header file is included by other header files there is a possibility that it can be included more than once and create errors.
    • The most common solution to this issue is called a include guard.
    • Simply looks like this: 

                #ifndef _INCLUDEA_H
            #define _INCLUDEA_H_

            //includeA.h header contents

            #endif /* _INCLUDEA_H_

    • A less common but simpler solution is to use #pragma once
    • Two disadvantages to this solution however is that the compiler may be fooled by symbolic links, and it may not be supported by all compilers.

                #pragma once
            //includeA.h header contents
