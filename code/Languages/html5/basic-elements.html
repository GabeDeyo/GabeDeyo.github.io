Basic Verbage

  • • HTML is a markup language. HyperText Markup Language.
    • HTML is used to mark up content in order to explain what that content is and how it relates to other content on the page.

  • • HTML uses tags which identify specific types of content.
    • Elements refer to tags and their contents.

  • • There are basically two types of attributes that can be assigned to elements.
      • Informative attributes give extra information about the element.
      • Functional attributes change the behavior of the element.

  • • Nesting allows one element to be placed inside of another as a child of it's parent element.
    • Some elements cannot nest inside others and child tags must be closed before their parents tags.

Document Structure

  • Basic Structure

                <!doctype html>
            
            <html lang='en'>
            <head>
                <meta charset='utf-8'>
            
                <title>HTML5 Standard Template</title>
                <meta name='description' content='HTML5 Standard Template'>
                <meta name='author' content='website'>
            
                <link rel='stylesheet' href='css/styles.css?v=1.0'>
            </head>
            
            <body>
                <script src='script.js'></script>
            </body>
            </html>

  • • The DOCTYPE declaration tell the user agents which version of HTML it's using.

  • Head

    • The Head is where you'll place info about the document, references to scripts, styles, linkes, etc., and additional instructions to the browser about how the document should be rendered.
    • What can go in the head?
      • Title - <title>The Title of the Page</title>
      • Metadata - Declares the metadata of your page
      • Icons - <link rel="shortcut icon" href="facison.ico" type="image/x-icon"> adds a favicon (Tab Icon) to the page.
      • CSS Links - always sit in the head to help render the elements first
      • Language - <html lang="en-US"> sets the primary language of the document.

  • Meta Attributes

    • When declaring metadata with the meta tag, the following structure is the standard syntax:
    <meta UNIQUE_ATTR="..." content="...">

    Attribute: charset - specifies the character encoding for the HTML document
    • Value: character_set
      • UTF-8 - Character encoding for Unicode
      • ISO-8859-1 - Character encoding for the Latin alphabet

    Attribute: http-equiv
    • Value: content-type - specifies the character encoding for the document
      • text/html
    • Value: default-style - specifies the preferred style sheet to use
      • "link to preferred stylesheet"
    • Value: refresh - defines a time interval for the document to refresh itself (!Using refresh causes a failure in W3C's Accessiblity Guidelines)
      • some integer

    Attribute: name - names a specific piece of metadata
    • Value: application-name - specifies the name of the Web application that the page represents
    • Value: author - specifies the name of the author of the document
    • Value: description - specifies a description for the page
    • Value: generator - specifies on of the software packages used to generate the document
    • Value: keywords - specifies a comma separates list of keywords relevant to the page (helps seo)
    • Value: viewport - controls the viewport
      • "width=device-width" - sets the width of the page to follow the screen-width of the device
      • "initial-scale=1.0" - sets the initial zoom level when the page is first loaded by the browser.

  • Body

    • The Body simply holds all of the documents page content.
    • The standard structure places the page content at the top of the body.
    • Then script links are placed below the page content so that the page can render before any interaction can happen.
