Creating C# Delegates

  • • Delegates are analogous to C++ function pointers, or JavaScript's callbacks.
    • Delegates are type safe, so the C# compiler checks the function signatures of delegates at runtime. This helps prevent common errors, like passing wrong arguments or getting wrong results back.

  • • Delegates are used to define what are called callback functions. These allow you to pass functions around as if they were objects themselves.
    • So the class that you pass the function to, can call the function itself. This is helpful when we need to dynamically switch the delegate at runtime.

  • • Delegates are created using the delegate keyword, along with a function signature that defines the type of function that the delegate will refer to.
    • Using the function signature will allow the compiler to check to make sure that the delegate being called has the right number of types and arguments, making them type safe.
    Example 

                public delegate int MyDelegate(int i, string s);

            int MyFunction(int i, string s) {
                // delegate implementation goes here
            }

            // Somewhere in the app code
            MyDelegate f = MyFunc;
            // later, call the delegate
            int result = f(250, "Hello World");

  • Real World Example 1

                namespace BasicDelegates {
                // Declare the delegate type
                public delegate string MyDelegate(int arg1, int arg2);

                class Program {
                    // Insert Delegates Here
                    static string func1(int a, int b) {
                        return (a + b).ToString();
                    }

                    static string func2(int a, int b) {
                        return (a * b).ToString();
                    }

                    static void Main(string[] args) {
                        // Declare delegate here
                        MyDelegate f = func1;
                        Console.WriteLine("The number is: " + f(10, 20)); // The number is: 30
                        f = func2;
                        Console.WriteLine("The number is: " + f(10, 20)); // The number is: 200
                    }
                }
            }

  • Real World Example 2

    • You can also use instance methods of classes as delegates. Let's change the first example a little:

                nnamespace BasicDelegates {
                // Declare the delegate type
                public delegate string MyDelegate(int arg1, int arg2);

                class MyClass {
                    // Delegates can be bound to instance members as well as
                    // static class functions
                    public string instanceMethod1(int arg1, int arg2)
                    {
                        return ((arg1 + arg2) * arg1).ToString();
                    }
                }

                class Program {
                    static void Main(string[] args) {
                        MyClass mc = new MyClass();
                        f = mc.instanceMethod1;
                        Console.WriteLine("The number is: " + f(10, 20)); // The number is: 300
                    }
                }
            }

Anonymous Delegates

  • • In addition to explicitly declaring and naming your Delegate functions, they can also be created as anonymous functions.
    • This is useful when your Delegate is a one-off function, or a function small enough to be declared in-line so that the code is more readable.

  • • When declaring the delegate, it's the same way as declaring an explicit delegate. 

    public delegate int MyDelegate(int arg1, int arg2);

    • However, instead of writing a named function, and then assigning that function to a variable of the Delegate type, we simply declare the function directly in line with no name.

                {
                ...
                // Somewhere in app code
                MyDelegate f = delegate(int arg1, int arg2) {
                    //implementation code goes here, right inline
                    return (arg1 + arg2).ToString();
                };

                // later, call the delegate
                Console.WriteLine("The number is: " + f(10, 20)); // The number is: 30
            }

  • • Functionally this is no different than declaring a regular function, and then doing the assignment in two different steps.
    • It's simply a little more convenient to do it this way when the Delegate is relatively simple.
    • Anonymous Delegates can usually be found being used with Event Handlers, which we'll see later.

Composable Delegates

  • • An interesting feature of delegates is that they are composable. Which means you can chain delegates together.
    • This allows delegates to be called in order that they were added to the chain.

  • • Unhandled exceptions will skip all remaining delegates.
    • Only the last return value will be returned to the calling function
    • To pass results from one delegate to another, use ref variables.

  • Chaining/Unchaining Example

                namespace ComposableDelegates {
                // declare the delegate type
                public delegate void MyDelegate(int arg1, int arg2);

                class Program {

                    static void func1(int arg1, int arg2) {
                        string result = (arg1 + arg2).ToString();
                        Console.WriteLine(result);
                    }
                    static void func2(int arg1, int arg2) {
                        string result = (arg1 * arg2).ToString();
                        Console.WriteLine(result);
                    }

                    static void Main(string[] args) {
                        MyDelegate f1 = func1;
                        MyDelegate f2 = func2;
                        //Chained delegate
                        MyDelegate f1f2 = f1 + f2;

                        // call each delegate and then the chain
                        f1(10, 20); // 30
                        f2(10, 20); // 200
                        f1f2(10, 20); // 30 \n 200

                        //Unchain f1 from chained delegate
                        f1f2 -= f1;
                        f1f2(20, 20); // 400
                    }
                }
            }

  • • A more advanced composable delegate strategy is using reference arguments.
    • These allow the function to change the value of that argument and have it take effect outside of the function.
    • This is where we'll use this ability to pass changes from one delegate to the next.

  • Reference Chain Example

                namespace ComposableDelegates2 {
                // Using reference parameters enables results to be passed
                // along from one delegate to the next
                public delegate void MyDelegate(int arg1, ref int arg2);

                class Program {

                    static void func1(int arg1, ref int arg2) {
                        string result = (arg1 + arg2).ToString();
                        arg2 += 20; // arg2 is a ref parameter, so this will change it's value
                        Console.WriteLine(result);
                    }

                    static void func2(int arg1, ref int arg2) {
                        string result = (arg1 * arg2).ToString();
                        Console.WriteLine(result); 
                    }

                    static void Main(string[] args) {
                        int a = 10, b = 10;
                        MyDelegate f1 = func1;
                        MyDelegate f2 = func2;
                        MyDelegate combined = f1 + f2;

                        Console.WriteLine("The value of b is: {0}", b);
                        combined(a, ref b);
                        Console.WriteLine("The value of b is: {0}", b);
                    }
                }
            }
