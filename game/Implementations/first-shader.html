Shader Overview

  • Note - The starting project should features a massive pink man.
      • The pink effect is actually "Unity's Pink Error" shader.
      • It is displayed anytime anything goes wrong with the rendering process.

  • Rendering Process

    A 3D Model contains Vertices, Vertex Colors, UV data, and Normals.

    A GameObject with a Mesh Renderer has a Material assigned.
      • This Material uses a Shader and holds textures, colors, and property values.

    The Shader uses data from the 3D model and from the Material to draw pixels to the screen based on it's CG / HLSL Code.

  • Shader

    A shader is a computer program that is used to do shading during the rendering proces.
      • Shading: The production of appropriate levels of light, darkness, and color within an image, or, in the modern era, also to produce special effectts or do video post-processing (like image-effects).

    A shader is a program specifically made to run on a GPU.
      • It is, ultimately, what draws the triangles of your 3D models.

  • Types of Shaders in Unity

    Surface Shaders in Unity are a code generation approach that makes it much easier to write lit shaders than using low level vertex/pixel shader programs.

    Unlit Shaders do not interact with Unity Lights, useful for special effects. The type created in this implementation.

    Image Effect Shaders are typically a postprocessing effect that read the source image, do some calculations on it, and render the result into the provided destination (e.g using Graphics.Blit)

    Compute Shaders are programs that run on the graphics card, outside of the normal rendering pipeline. They can be used for massively parallel GPGPU algorithms, or to accelerate parts of game rendering.

Anatomy of an Unlit Shader

  • Open up the Hologram Shader.

    It will automatically have a standard setup, just like the MonoBehavior scripts.

    The language shaders are written in is called "ShaderLab".

                // Initial Shader Setup

            Shader "Unlit/Hologram"
            {
                Properties
                {
                    _MainText ("Texture", 2D) = "white" {}
                }
                SubShader
                {
                    Tags { "RenderType"="Opaque" }
                    // level of detail
                    LOD 100

                    Pass
                    {
                        CGPROGRAM
                        #pragma vertex vert
                        #pragma fragment frag
                        // make fog work
                        #pragma multi_compile_fog

                        #include "UnityCG.cginc"

                        struct appdata
                        {
                            float4 vertex : POSITION;
                            float2 uv : TEXCOORD0;
                        };

                        struct v2f
                        {
                            float2 uv : TEXCOORD0;
                            UNITY_FOG_COORDS(1)
                            float4 vertex : SV_POSITION;
                        }

                        sampler2D _MainTex;
                        float4 _MainTex_ST;

                        v2f vert (appdata v)
                        {
                            v2f o;
                            o.vertex = UnityObjectToClipPos(v.vertex);
                            o.uv = TRANSFORM_TEX(v.uv, _MainTex);
                            UNITY_TRANSFER_FOG(o,o.vertex);
                            return o;
                        }

                        fixed4 frag (v2f i) : SV_Target
                        {
                            // sample the texture
                            fixed4 col = tex2D(_MainTex, i.uv);
                            // apply fog
                            UNITY_APPLY_FOG(i.fogCoord, col);
                            return col;
                        }
                        ENDCG
                    }
                }
            }

  • Shader

    Defines a new shader in a specific folder.

    We can change the value to "Unlit/SpecialFX/Cool Hologram"

  • Properties

    Used to define public variables

    We can change the _MainText value to ("Albedo Texture", 2D) since this is the main albedo color.

    The "Albedo Texture" serves as the display name, and the "2D" serves as the value.

    Note - No semicolons

  • SubShader

    Contains instructions for Tags on how to setup the renderer and the Pass information.

    There can be multiple SubShader definitions.
      • Could be used to define separate ways to render for mobile, dektop, platforms, etc.

    In the initial unlit shader, the SubShader contains two struct definitions. (appdata, v2f)
      • These are passed to two functions (v2f vert (appdata v), fixed4 frag (v2f i) : SV_Target)

  • Unlit Shader

    Vertex Function - Takes the shape of model, potentially modifies it.

    Fragment Function - Applies color to the shape output by the vert function.

    Property Data: Colors, textures, values set by user in the Inspector.

    Mesh -> Vertex Function (Property Data) -> Fragment Function (Property Data) -> Image

Vertex Function

  • We'll be looking initially at the Pass block.

    The Pass block starts off with some definitions (preprocessor directives).
      • Defines a vertex function called vert.
      • Defines a fragment function called frag.
      • We'll take out the fog definition which resides throughout the SubShader.

    The #include "UnityCG.cginc" brings in that class as a depencency at compile time.

    Two struct are created (appdata, v2f (vert 2 frag)) that contain their own struct data.
      • After the colon, we define semantic binding. This tells the render how we're using the data.

    The struct data is then being passed to two functions.

  • Coordinate Systems

    Local space (or Object space)

    World space

    View space (or Eye space)

    Clip space

    Screen space

  • vert Function

                v2f vert (appdata v)
            {
                // Create new v2f struct o
                v2f o;

                // Set the o vertex to the apporpriate coordinate space (screen space)
                o.vertex = UnityObjectToClipPos(v.vertex);

                // Transform the texture onto the model
                o.uv = TRANSFORM_TEX(v.uv, _MainText);

                // Return the built up model
                return o;
            }

Fragment Function and Color Tint

  • frag Function

                fixed4 frag (v2f i) : SV_Target
            {
                // sample the texture (colors) from the MainTex and uv data
                fixed4 col = tex2D(_MainTex, i.uv);

                // return Screen Space buffer information
                return col;
            }

  • Tint Color

    In Properties, we'll add some variables.
      • _TintColor("Tint color", Color) = (1,1,1,1)

    Declare these variables in the Pass block.
      • float4 _TintColor;

    Use this variable by adding it to the col variable.
      • fixed4 col = text2D(_MainTex, i.uv) + _TintColor;

    In the Shader inspector, set this color to a nice dark blue.

Transparency

  • Head to Tags block in SubShader.
      • Set the RenderType to "Transparent" rather than Opaque.
      • In front the the RenderType, set "Queue" to "Transparent" as well.

    Below the LOD, add these configurations:
      • ZWrite Off - Don't render to depth buffer
      • Blend SrcAlpha OneMinusSrcAlpha - Render in order and blend using alpha channel

    Add a public property to Properties:
      • _Transparency("Transparency", Range(0.0, 0.5)) = 0.25

    Declare Transparency in Pass block.
      • float _Transparency;

    Head to the frag function to add a line below the color declaration.
      • Add: col.a = _Transparency // Sets color alpha channel to transparency

  • Updated Shader Script

                Shader "Unlit/SpecialFX/Cool Hologram"
            {
                Properties
                {
                    _MainTex ("Albedo Texture", 2D) = "white" {}
                    _TintColor("Tint Color", Color) = (1,1,1,1)
                    _Transparency("Transparency", Range(0.0,0.5)) = 0.25
                }
            
                SubShader
                {
                    Tags {"Queue"="Transparent" "RenderType"="Transparent" }
                    LOD 100
            
                    ZWrite Off
                    Blend SrcAlpha OneMinusSrcAlpha
            
                    Pass
                    {
                        CGPROGRAM
                        #pragma vertex vert
                        #pragma fragment frag
            
                        #include "UnityCG.cginc"
            
                        struct appdata
                        {
                            float4 vertex : POSITION;
                            float2 uv : TEXCOORD0;
                        };
            
                        struct v2f
                        {
                            float2 uv : TEXCOORD0;
                            float4 vertex : SV_POSITION;
                        };
            
                        sampler2D _MainTex;
                        float4 _MainTex_ST;
                        float4 _TintColor;
                        float _Transparency;
                        
                        v2f vert (appdata v)
                        {
                            v2f o;
                            o.vertex = UnityObjectToClipPos(v.vertex);
                            o.uv = TRANSFORM_TEX(v.uv, _MainTex);
                            return o;
                        }
                        
                        fixed4 frag (v2f i) : SV_Target
                        {
                            // sample the texture
                            fixed4 col = tex2D(_MainTex, i.uv) + _TintColor;
                            col.a = _Transparency;
                            return col;
                        }
                        ENDCG
                    }
                }
            }

Displacing Vertices and Clipping Pixels

  • In this section, we want to be able to throw away certain colors under certain conditions.

    Add a property to the Properties block:
      • _CutoutThresh("Cutout Threshold", Range(0.0, 0.1)) = 0.2

    Remeber to declare this property in the Pass block:
      • float _CutoutThresh;

    Head to the frag function and add this line below the Transparency line.
      • clip(col.r - _CutoutThresh);
      • Could be understood better as (don't implement): if (col.r < _CutoutThresh) discard;

  • In this section, we want to displace verticies to create a glitch effect.

    Add these properties to the Properties block:
      • _Distance("Distance", Float) = 1
      • _Amplitude("Amplitude", Float) = 1
      • _Speed("Speed", Float) = 1
      • _Amount("Amount", Range(0.0, 1.0)) = 1

    Then declare all of the properties in the Pass block:
      • float _Distance;
      • float _Amplitude;
      • float _Speed;
      • float _Amount;

    Use these now in the vert function after the o declaration:
      • v.vertex.x += sin(_Time.y * _Speed * v.vertex.y * _Amplitude) * _Distance * _Amount;

    In Scene view, mess with the values until there is a sweet effect.

  • HoloMan Glitcher

    This is the script that handles the random glitching look.

                public class HoloManGlitcher : MonoBehaviour {

                public float glitchChance = 0.1f;

                private Renderer holoRenderer;
                private WaitForSeconds glitchLoopWait = new WaitForSeconds(0.1f);
                private WaitforSeconds glitchDuration = new WaitForSeconds(0.1f);

                void Awake() {
                    holoRenderer = GetComponent<Renderer>();
                }

                IEnumerator Start() {
                    while (true) {
                        float glitchText = Random.Range(0f, 1f);

                        if (glitchTest <= glitchChance) {
                            StartCoroutine (Glitch ());
                        }
                        
                        yield return glitchLoopWait;
                    }
                }

                IEnumerator Glitch() {
                    glitchDuration = new WaitForSeconds(Random.Range(0.05f, 0.25f));
                    holoRenderer.material.SetFloat ("_Amount", 1f);
                    holoRenderer.material.SetFloat ("_CutoutThresh", 0.29f);
                    holoRenderer.material.SetFloat ("_Amplitude", Random.Range(100, 250));
                    holoRenderer.material.SetFloat ("_Speed", Random.Range(1, 10));

                    yield return glitchDuration;

                    holoRenderer.material.SetFloat ("_Amount", 0f);
                    holoRenderer.material.SetFloat ("_CutoutThresh", 0f);
                }
            }
