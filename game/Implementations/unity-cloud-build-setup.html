Introduction

  • Unity Cloud Build watches your source code and automatically compiles, deploys, and tests new changes.

    Git is a popular version control system that allows you to track changes to your project and revert them when needed.

    Together, they make the perfect toolset for speeding up Unity game development.

    In this section, we'll look at setting up Unity Cloud Build projects, tag builds, push changes via Git to Bitbucket, fix bugs, and manage project changes.

    Later we'll see how to optimize your workflow to get the most out of the toolset and compile games across multiple platforms: WebGL, desktop, and mobile.

Repo Setup

  • Git Setup

    Open GitBash in the project you want to setup in UCB.

                git init

  • User Setup

    Git needs to know your username and email.

                git config --global user.email "gabedeyo@gmail.com"
            git config --global user.name "Gabe Deyo"

  • .gitignore

    Exclude files getting pushed to the repo using the .gitignore file.

    Save .gitignore into the root of your project folder.

                /[Ll]ibrary/
            /[Tt]emp/
            /[Oo]bj/
            /[Bb]uild/
            /[Bb]uilds/

            *.csproj
            *.unityproj
            *.sln
            *.suo
            *.tmp
            *.user
            *.userprefs
            *.pidb
            *.boo

  • Add Changes

    Commit all changes

                git add .
            //or
            git add --all

    Commit specific changes

                git add boner.txt

    Check git status

                git status

  • Commit Changes

    Make sure all changes that you want committed are added with git status.

                git commit -m "Commit Message"

  • Bitbucket Repo

    Create a new repo in your Bitbucket account.

    Link your local project and Bitbucket repo:

                git remote add origin git@bitbucket.org:accountname/reponame.git

    Head to SourceTree and add this project path.

    Use source tree to push your commit to the repo.
