Awake & Start

  • Awake
      • Called first, even if script is NOT enabled.
      • Used for references between scripts and initializations.

  • Start
      • Called after Awake, immediately before the first Update. But only if script is enabled.

  • Example
      An enemy character could enter the game. The ammo count would be set in Awake,
      while the ability to shoot would be called in Start.

  • * Only called once in the entire lifetime of a GameObject.

Render specic layers using Camera Culling Mask

  • Use camera culling mask to choose which layers the game view will render.

  • Note* these objects will be rendered in the scene view but not the game view.

Limit physics interactions using Layers and Bitmasks

  • public Collider[] neighbors; // Get a list of colliders in an Array
    public float distance = 10; // Start distance for it to look inside, based on level scale
    public int waypointMask = 1 << 8; //Place value of 1(true) as the value of layer 8.

  • void Start(){
     getNeighbors(); // Find surrounding waypoints
    }

  • // Get nearby waypoints based on a sphere of distance that increases until it has 3 neighbors. This object will be in the list, and we want at least 2 other connected neighbors.

    void getNeighbors(){
     while(neighbors.Length < 3){
      //Search in a sphere of radius distance for other colliders
      neighbors = Physics.OverlapSphere(this.transform.position, distance, waypointMask);
      //If we haven't found enough neighbors, increase the search size
      distance += 10;
     }
    }

Calculating the Value of a Bitmask and Combining Bitmasks

  • Set individually layers appropriatly. In example case - NorthWaypoints(9), SouthWaypoints(10)

  • Take 2 to the power of the layer. This is the number of the bitmask.

  • * Used to control how our layers interact and how physics interacts between layers.

Render Modes

  • Select the specific render mode in Scene View tool bar

  • 1. RGB, normal view, seeing all colors

  • 2. Alpha, set everything to black or white.
    Helpful for seeing places where you can see through the level.

  • 3. Overdraw, shows you a sillouette of everything being overdrawn.
    Lighter or more saturation means more overdraw.
    Helpful for seeing things that should be culled.

  • 4. Mipmaps, show resolution ideals for textures.
    When you are further away, mipmaped textures will drop in resolution.
    Red shows too much resolution, blue is not enough resolution. White is okay.

Cube Maps

  • Use cube maps to make reflective materials.

  • Create new Material and set it's shader to be reflective/diffuse

  • Create a Cubemap in project view. Name Material and Cubemap to "Sky"

  • Apply Cubemap textures to the cubemap. (Get from somewhere)
    Apply this Cubemap to the Sky Material.

  • Set tone to something less blown out or more visual with main color of material.

  • IF using this as a skybox,
    set shader to RenderFX/Skybox Cubed

  • In Edit/RenderSetting, plug in that skybox

Unity Cloth

  • For testing, bring in something to collide with like a sphere or cube.

  • Bring in a cloth which helps it simulate in Unity.
    The cloth renderer helps us see the cloth.

  • *Issues may arise if items colliding with cloth are too small.
    They will rip right through.

  • Worth experimenting with all of the properties of the cloth.
